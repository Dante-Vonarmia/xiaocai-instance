# 云鹤AI接入采购中国小程序对接技术要求

文档状态：待采购中国侧确认与补充  
日期：2026-06-11

## 1. 文档目的

本文档用于定义采购中国小程序内嵌云鹤AI的最小安全接入方案，并明确云鹤AI侧开发承接范围、采购中国侧需提供的对接能力与联调资料。

本方案采用：

```text
URL 授权参数 + 短期一次性登录凭证 + 服务端校验 + 云鹤AI本地登录态 + 外部访问拦截
```

## 2. 接入目标

云鹤AI当前已有线上地址：

```text
https://yunheai.cloud/
```

采购中国小程序通过 `web-view` 内嵌打开该地址。采购中国已登录用户进入云鹤AI时，采购中国侧生成通过 URL 参数传递的短期一次性登录凭证，云鹤AI后端通过服务端接口校验登录凭证后建立本地登录态。

目标效果：

```text
采购中国小程序已登录用户 -> 可进入云鹤AI
外部直接访问 yunheai.cloud -> 被拦截
无效或过期登录凭证 -> 被拦截
```

## 3. 接入范围

本期范围：

- 使用现有地址 `https://yunheai.cloud/`。
- 采购中国小程序通过 `web-view` 打开云鹤AI。
- 采购中国生成短期一次性登录凭证。
- 云鹤AI后端校验登录凭证。
- 校验成功后，云鹤AI建立本地登录态。
- 云鹤AI展示当前登录用户。
- 云鹤AI拦截外部直接访问。
- 云鹤AI后端 API 增加登录态保护。

非本期范围：

- 不新增独立入口地址。
- 不接采购中国数据库。
- 不同步采购中国会员表。
- 不同步供应商、活动、订单等数据。
- 不重做采购中国登录系统。
- 不决定采购中国小程序内部栏目位置。

## 4. 术语与推荐命名

本方案推荐 URL 参数名为：

```text
login_ticket
```

备选参数名：`sso_ticket`、`auth_code`。如采购中国已有统一登录、SSO 或 OAuth 接入规范，可优先沿用其既有参数名和接口命名。

完整术语表见：`docs/caigou-china-miniapp-integration-terms.md`。

## 5. 采购中国侧需提供内容

采购中国侧需提供以下资料或能力：

| 类别 | 需提供内容 |
| --- | --- |
| web-view 准入 | 确认小程序可打开 `https://yunheai.cloud/`；如需业务域名校验，请提供校验文件 |
| 登录凭证生成 | 生成系统、生成时机、有效期、是否一次性、是否绑定当前用户、是否绑定云鹤AI应用 |
| 登录凭证校验接口 | 接口地址、请求方式、请求参数、返回格式、错误码、测试环境地址 |
| 接口鉴权 | appId、签名算法、timestamp 规则、nonce 规则、IP 白名单要求、测试密钥交付方式 |
| 用户字段 | 返回用户ID、名称/昵称、状态，以及可选的手机号/openid/unionid |
| 测试资料 | 测试小程序入口、测试账号、有效登录凭证、无效登录凭证、过期登录凭证、已使用登录凭证样例 |

生产密钥不应通过普通聊天群直接传递，应通过安全渠道交付。

## 6. 总体接入链路

```text
采购中国小程序已登录用户
  -> 点击云鹤AI
  -> 采购中国后端生成一次性登录凭证
  -> 小程序 web-view 打开 https://yunheai.cloud/?login_ticket={auth_credential}
  -> 云鹤AI后端调用采购中国校验接口
  -> 校验成功后创建云鹤AI本地登录态
  -> 清理 URL 中的授权参数
  -> 进入云鹤AI应用
```

外部访问链路：

```text
外部直接访问 https://yunheai.cloud/
  -> 无 URL 授权参数
  -> 无云鹤AI登录态
  -> 拦截
```

## 7. URL 约定

云鹤AI地址：

```text
https://yunheai.cloud/
```

小程序打开格式：

```text
https://yunheai.cloud/?login_ticket={auth_credential}
```

说明：`login_ticket` 为推荐 URL 参数名，最终参数名可由双方约定；`{auth_credential}` 为采购中国侧生成的短期一次性登录凭证。

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| `login_ticket` | 是 | 推荐 URL 授权参数名；最终参数名可由双方约定。参数值为采购中国侧生成的短期一次性登录凭证 |

URL 授权参数值不应包含手机号、用户ID、会员等级等明文业务信息，也不应是长期有效 token。

## 8. 登录凭证要求

采购中国侧通过 URL 参数传递的登录凭证应满足以下要求：

| 项目 | 要求 |
| --- | --- |
| 类型 | 短期一次性登录凭证 |
| 生成时机 | 用户点击云鹤AI入口时生成 |
| 有效期 | 建议 1-5 分钟 |
| 使用次数 | 建议一次性使用，校验成功后立即失效 |
| 随机性 | 随机、不可预测，不能由用户ID或手机号直接拼接 |
| 绑定对象 | 当前采购中国小程序登录用户 |
| 绑定应用 | 建议绑定云鹤AI应用标识，如 `yunhe_ai` |
| 明文信息 | 不在 URL 中传递手机号、用户ID、会员等级等明文字段 |

## 9. 登录凭证校验接口要求

推荐由采购中国提供服务端校验接口，云鹤AI后端拿到 URL 授权参数值后调用该接口换取当前用户信息。

请求方向：

```text
云鹤AI后端 -> 采购中国后端
```

该接口不应由浏览器前端直接调用。

### 8.1 接口鉴权

采购中国侧的登录凭证校验接口建议增加服务端调用鉴权，避免任意第三方调用。

推荐至少支持以下一种或多种方式：

```text
appId + appSecret 签名
timestamp + nonce 防重放
云鹤AI服务器出口 IP 白名单
HTTPS
```

推荐请求格式：

```json
{
  "appId": "yunhe_ai",
  "credential": "abc123",
  "timestamp": 1710000000,
  "nonce": "random_string",
  "signature": "HMAC_SHA256(appId + credential + timestamp + nonce, appSecret)"
}
```

具体签名算法可由采购中国侧按现有安全规范确定。

### 8.2 返回格式

成功返回示例：

```json
{
  "valid": true,
  "user": {
    "id": "U10001",
    "name": "张三",
    "mobile": "138****8888",
    "status": "active",
    "openid": "optional_openid",
    "unionid": "optional_unionid"
  }
}
```

失败返回示例：

```json
{
  "valid": false,
  "error": "CREDENTIAL_EXPIRED",
  "message": "登录凭证已过期"
}
```

建议错误码：

| 错误码 | 说明 |
| --- | --- |
| CREDENTIAL_MISSING | 登录凭证缺失 |
| CREDENTIAL_INVALID | 登录凭证无效 |
| CREDENTIAL_EXPIRED | 登录凭证已过期 |
| CREDENTIAL_USED | 登录凭证已使用 |
| USER_DISABLED | 用户不可用 |
| SIGNATURE_INVALID | 签名无效 |
| APP_UNAUTHORIZED | 应用未授权 |
| VERIFY_FAILED | 校验服务异常 |

## 10. 用户字段要求

校验成功后，采购中国至少返回以下字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| id | 是 | 采购中国用户/会员唯一ID |
| name | 是 | 用户名称或昵称，用于云鹤AI页面展示 |
| status | 是 | 用户/会员状态，用于判断是否允许进入 |
| mobile | 否 | 手机号，可脱敏 |
| openid | 否 | 微信 openid，如已有可返回 |
| unionid | 否 | 微信 unionid，如已有可返回 |

云鹤AI侧仅保存登录识别、页面展示和审计所需的最小用户信息。

## 11. 云鹤AI侧开发承接

云鹤AI侧需要完成以下能力：

| 模块 | 说明 |
| --- | --- |
| URL 授权参数读取 | 支持从 `https://yunheai.cloud/?login_ticket={auth_credential}` 读取 URL 授权参数 |
| 登录凭证校验 | 云鹤AI后端调用采购中国登录凭证校验接口 |
| 登录态建立 | 校验成功后创建云鹤AI本地登录态 |
| 用户映射 | 保存采购中国用户ID与云鹤AI本地用户的最小映射 |
| 当前用户展示 | 页面展示当前登录用户名称或昵称 |
| URL 清理 | 成功登录后清理 URL 中的授权参数 |
| 外部拦截 | 无 URL 授权参数且无登录态时不进入业务页面 |
| API 鉴权 | 后端 API 未登录返回 401 或 403 |
| 日志 | 记录入口访问、校验成功、校验失败、拦截、API 未授权等事件 |

云鹤AI登录态中至少包含：

```text
external_user_id
source = caigou_china
display_name
member_status
last_login_at
```

默认拦截提示：

```text
请从采购中国小程序进入云鹤AI服务
```

## 12. 联调验证方式

接口验证：先验证采购中国登录凭证校验接口是否可用。

```bash
curl -X POST "https://{采购中国测试接口域名}/api/auth/verify-credential" \
  -H "Content-Type: application/json" \
  -d '{"appId":"yunhe_ai","credential":"TEST_CREDENTIAL","timestamp":1710000000,"nonce":"n_123456","signature":"signature_value"}'
```

云鹤AI访问验证：

```text
https://yunheai.cloud/?login_ticket=TEST_CREDENTIAL
```

小程序链路验证：从采购中国测试小程序进入云鹤AI，确认用户身份识别正确，外部直接访问被拦截。

## 13. 验收标准

| 场景 | 预期结果 |
| --- | --- |
| 小程序携带有效登录凭证打开云鹤AI | 正常进入云鹤AI |
| 云鹤AI页面展示当前用户 | 用户信息正确 |
| 登录成功后清理 URL 中的授权参数 | 地址栏不再保留登录凭证 |
| 外部直接访问 `https://yunheai.cloud/` | 被拦截 |
| 无 URL 授权参数且无登录态访问 | 被拦截 |
| 无效登录凭证访问 | 被拦截 |
| 过期登录凭证访问 | 被拦截 |
| 已使用登录凭证再次访问 | 被拦截，若登录凭证为一次性 |
| 签名错误请求校验接口 | 被拒绝 |
| 未登录直接调用云鹤AI API | 返回 401 或 403 |
| 登录态有效期内刷新页面 | 可继续使用 |

## 15. 结论

本次对接采用：

```text
现有云鹤AI地址
+ 采购中国小程序 web-view
+ 短期一次性登录凭证
+ 服务端校验接口
+ 接口签名或等价鉴权
+ 云鹤AI本地登录态
+ URL 授权参数清理
+ 外部访问拦截
```

采购中国侧核心需提供：

```text
小程序打开 https://yunheai.cloud/?login_ticket={auth_credential}
登录凭证生成规则
登录凭证校验接口或等价校验机制
校验接口鉴权方式
校验成功后的用户信息
测试入口和测试账号
```
