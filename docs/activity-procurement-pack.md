# 活动采购 Domain Pack（P0）

## 适用范围
- 企业活动相关采购：线下活动执行、物料与搭建、会务支持、活动传播协同

## 业务边界
- 负责：需求结构化、活动分析、供应商筛选与评分、RFX动作建议、标准产物输出
- 不负责：活动创意最终拍板、线下执行管理系统、合同法务定稿

## 字段分层

## 必填字段
- 项目名称
- 采购目的
- 使用场景（活动类型/规模）
- 交付地点
- 交付时间
- 预算金额
- 数量/规模口径（人次、场次、物料规模）
- 一级品类/二级品类

## 推荐字段
- 活动目标人群
- 关键里程碑
- 验收标准
- 风险偏好（稳妥优先/创意优先）
- 供应商区域偏好
- 历史同类项目参考

## 可选字段
- 对标活动案例
- 指定合作禁限名单
- ESG/环保偏好
- 增值服务偏好

## 缺失字段补问逻辑
1. 先补齐“预算、交付时间、交付地点、规模口径”四个执行约束
2. 再补齐“验收标准、风险偏好、区域偏好”
3. 若仍缺关键字段，仅输出缺口清单与下一步问题，不输出完整推荐

## 分析输出七段式骨架
1. 项目理解和核心需求
2. 市场现状和分析
3. 成本结构分析
4. 项目风险分析
5. 采购策略分析
6. 供应商选择建议
7. 项目实施计划与执行建议

## 供应商筛选关键维度
- 区域
- 注册资金
- 成立时长
- 员工规模
- 资质
- 服务模式
- 技术能力
- 本地化能力
- 质量保障
- 安全保障
- 环保能力
- 商务条件
- 响应速度
- 案例经验
- 增值服务

## 采购动作建议（RFI / RFP / RFQ / RFB）
- RFI：需求尚不稳定、先摸清市场与能力版图
- RFP：方案能力与执行能力是核心，需比方案
- RFQ：需求规格已清晰，重点比价格与交付条件
- RFB：条件标准化且需公开/规范化竞标流程时使用

## 应输出的 artifact
- 活动需求梳理摘要
- 结构化需求单（活动版）
- 七段式需求分析文档
- 供应商 longlist / shortlist
- 供应商评分表（活动版）
- RFX动作建议单
- 风险提示单与澄清问题单

## 输入 / 输出 / 依赖 / 暂不做

### 输入
- 活动采购需求与约束字段

### 输出
- 活动场景可执行采购建议与标准产物

### 依赖
- supplier-evaluation-framework
- rfx-routing-spec
- artifact-output-spec

### 暂不做
- 活动现场执行排期系统化管理
