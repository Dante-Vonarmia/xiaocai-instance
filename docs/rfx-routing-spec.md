# RFX Routing Spec（P0）

## 目标
将 RFI / RFP / RFQ / RFB 的选择逻辑固化为实例规则，支持可解释采购动作建议。

## 输入参数
- 项目类型
- 采购目标（摸底/比方案/比价格/规范招标）
- 预算规模（区间）
- 约束条件（交期、合规、技术复杂度、不确定性）
- 需求完整度（字段完备程度）

## 判断规则（简版）
1. **需求不稳定 + 目标是摸底** -> RFI 优先
2. **需求目标导向方案质量、执行方法差异明显** -> RFP 优先
3. **规格清晰 + 可比价 + 约束明确** -> RFQ 优先
4. **规则化、公开化、强合规竞标要求明显** -> RFB 优先

## 典型场景例子
- 活动方案尚未定型，先看市场能力：RFI
- 活动执行需综合方案与案例比选：RFP
- 礼品规格已确定，仅比较价格和交付：RFQ
- 预算体量大且需规范招采流程：RFB

## 每种动作适用时机
- RFI：前期探索、需求边界不清
- RFP：中前期，需比较方法与方案质量
- RFQ：中后期，规格明确，追求高效比价
- RFB：规则与流程导向，强调正式竞标

## 输出建议结构
- 推荐动作类型（RFI/RFP/RFQ/RFB）
- 推荐理由（对应输入参数）
- 前置缺口（当前缺什么）
- 备选动作（次优方案）
- 下一步执行建议

## 与需求梳理 / 供应商筛选 / 采购决策关系
- 需求梳理：提供 routing 所需字段基础
- 供应商筛选：根据动作类型决定候选池策略与评估重点
- 采购决策：将 routing 输出作为决策依据之一，不替代人工审批

## 输入 / 输出 / 依赖 / 暂不做

### 输入
- 结构化需求字段
- 风险与约束信息

### 输出
- 可解释 RFX 动作建议

### 依赖
- fields schema 完整度
- supplier evaluation framework

### 暂不做
- 自动生成最终法务级招标文件
