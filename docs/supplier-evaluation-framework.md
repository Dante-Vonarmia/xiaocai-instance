# 供应商评估框架（活动 + 礼品）

## 目标
形成从 longlist 到 shortlist 的统一评估方法，避免仅凭创意或报价做决策。

## 通用评估维度
- 基础合规：主体合法性、资质、风险与诉讼
- 履约能力：交付稳定性、质量保障、响应速度
- 商务条件：价格合理性、付款/发票/条款匹配
- 服务能力：项目协同、问题闭环、增值服务
- 案例经验：同类项目经验与结果证明

## 场景差异化维度

### 活动类
- 本地化保障能力
- 现场执行与安全保障
- 跨供应链协同能力
- 创意落地与执行一致性

### 礼品类
- 定制工艺与打样能力
- 量产一致性
- 包装与物流协同
- 环保/材质合规

## 硬门槛 vs 加分项

### 硬门槛（不满足即淘汰）
- 主体合法合规资料完整
- 场景必要资质满足
- 基本履约能力满足（交期/质量底线）
- 无重大红线风险

### 加分项
- 本地化能力强
- 同类头部案例
- 增值服务成熟
- 响应时效优于基线

## 活动类建议评分框架（示例）
- 合规与资质：20%
- 执行与履约能力：25%
- 质量/安全/环保保障：20%
- 商务条件：20%
- 案例与增值服务：15%

## 礼品类建议评分框架（示例）
- 合规与资质：20%
- 定制与打样能力：25%
- 量产质量与交付能力：25%
- 商务条件：20%
- 案例与服务协同：10%

## 红线风险
- 资质造假或关键合规信息缺失
- 重大诉讼/处罚且无合理解释
- 关键条款含糊（交期、质保、赔付）
- 明显低价但无法说明成本结构

## shortlist 推荐输出结构
- 供应商基础信息摘要
- 评分明细与总分
- 核心优势
- 核心风险
- 适配场景说明
- 推荐结论（入围/观察/淘汰）

## longlist -> shortlist 筛选逻辑
1. longlist 准入：按硬门槛初筛
2. 维度评分：按场景评分框架打分
3. 风险复核：剔除红线与高风险
4. shortlist 生成：输出排名 + 风险提示 + 建议动作

## 输入 / 输出 / 依赖 / 暂不做

### 输入
- 场景需求字段
- 供应商资料与证据

### 输出
- 可解释评分结果
- shortlist 推荐

### 依赖
- activity/gift domain pack 字段口径

### 暂不做
- 黑箱式自动评分（不可解释）
