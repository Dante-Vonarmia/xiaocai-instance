# xiaocai 文档层 Gap Analysis（本轮）

## 已有
- 已有 FLARE 与 instance 边界文档（架构层）
- 已有 domain standards 总体框架（字段、流程、模板等）
- 已有 requirements baseline 背景材料（字段与RFX基础）

## 缺失
- 缺少“xiaocai instance 是什么”的实例级定性文档
- 缺少聚焦 P0（活动+礼品）的场景收口文档
- 缺少 domain assets 全景映射（谁产出、谁消费）
- 缺少活动/礼品两个 pack 的正式定义
- 缺少活动与礼品统一的供应商评估正式框架
- 缺少实例级 RFX routing 规则文档
- 缺少 P0 产物输出标准文档
- 缺少文档对齐后的最小启动顺序文档

## 模糊
- 现有文档多为“原则层”，缺少“场景资产层”
- P0 与 P1 的优先级边界不够清晰
- 领域资产与后续开发入口的映射不够直接

## 冲突
- 部分历史描述容易把 instance 理解为“配置壳”或“泛化流程平台”
- 部分内容强调通用能力，但未明确采购场景落地资产优先级

## 不该保留的旧描述（处理原则）
- 任何把 xiaocai 写成通用聊天产品的表述
- 任何把采购语义写回 FLARE Core 的表述
- 任何“全品类一期全做”的暗示性表述
