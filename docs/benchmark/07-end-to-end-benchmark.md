# End-to-End Benchmark - 端到端性能测试

**测试编号**: BM-E2E-2026-02-11
**测试版本**: v1.0 (大纲)
**状态**: 📅 计划中

---

## 测试目标

验证小采 1.0 MVP 全链路性能：
- 用户输入 → API 响应 → Agent 执行 → 数据库 → 返回结果

## 测试场景

### 场景 1: 需求梳理流程

**用户操作**:
1. 输入需求："需要采购100台办公电脑，预算50万"
2. Agent 多轮对话（3-5 轮）
3. 生成需求文档

**测量指标**:
- 端到端延迟 (P50, P95, P99)
- 每轮对话延迟
- 资源消耗（CPU, Memory, DB Queries）

### 场景 2: 智能寻源流程

**用户操作**:
1. 输入查询："北京办公家具供应商"
2. Agent 搜索 + 匹配
3. 返回供应商列表

**测量指标**:
- 搜索延迟
- 匹配准确率
- 并发能力（10/50/100 用户）

## 测试工具

- **Locust** - 性能测试
- **Grafana** - 监控可视化
- **Prometheus** - 指标采集

## 性能目标

| 指标 | Phase 0 目标 | Phase 1 目标 |
|------|-------------|-------------|
| API 响应延迟 (P95) | < 3s | < 2s |
| Agent 执行延迟 (P95) | < 5s | < 3s |
| 并发用户数 | 10 | 100 |
| 错误率 | < 1% | < 0.1% |
