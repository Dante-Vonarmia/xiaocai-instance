# 需求基线（合同对齐）

## 目标
为以下三个模块建立一套可执行的统一基线：
1. 需求梳理
2. 需求分析
3. RFX策略（RFI/RFQ/RFP/RFB）

## 端到端流程
需求梳理 -> 需求分析 -> RFX策略 -> 导出 -> 归档

## 模块交付物
### 需求梳理
1. 结构化需求记录（必填字段完整）。
2. 缺失字段补录记录与编辑历史。

### 需求分析
1. 基于因子的分析报告（可编辑、可版本化）。
2. 可用于业务决策的结论摘要。

### RFX策略
1. 推荐RFX类型（RFI/RFQ/RFP/RFB）及理由。
2. 策略权重配置与可导出结果。
3. 与源需求关联的归档记录。

## 验收门槛
1. 字段完整性和字段含义通过业务确认。
2. 分析结构与因子逻辑通过业务确认。
3. RFX类型规则、权重逻辑、导出结果通过业务确认。

## 需要合伙人反馈的内容
1. 字段调整：新增/删除/重命名。
2. 必填等级调整：必填/条件必填/可选。
3. 需要补充的行业参考数据来源。
4. 三个模块各自的验收阈值。
