# 采购智能配置中心方案与排期

最后更新: 2026-05-18

## 1. 背景

当前 xiaocai 已经具备 requirement intake、字段收集、需求分析、寻源和产物生成的主链路规划。  
下一步需要让用户可以在设置页里维护更稳定的采购语义，包括具体品类、模板、资料库和 AI 生成规则。

本方案回答三个问题：

1. 设置页面是否需要支持品类与模板编辑。
2. 资料库能增强到什么程度。
3. 是否需要可管理的提示词或生成规则。

## 2. 产品判断

建议建设“采购智能配置中心”，而不是裸露的“大段提示词编辑器”。

原因：

- 只靠 prompt 不能保证每次完全一致输出。
- 稳定输出需要字段 schema、模板结构、校验规则和渲染边界共同约束。
- 用户需要管理采购知识与生成偏好，但不应承担维护系统 prompt 的风险。
- xiaocai 应承接采购产品语义，FLARE 只承接通用 runtime / execution 能力。

## 3. 能力边界

### 3.1 做什么

配置中心第一阶段包含四类能力：

1. 品类配置
   - 品类字段
   - 必填字段
   - 推荐追问
   - 分析维度
   - 寻源偏好

2. 模板管理
   - 需求澄清模板
   - 需求分析模板
   - RFI / RFQ / RFP 模板
   - 供应商对比模板
   - 邮件 / 文档产物模板

3. 资料库
   - 公司采购政策
   - 品类知识
   - 历史案例
   - 供应商资料
   - 示例文档

4. AI 规则管理
   - 全局生成规则
   - 品类专属规则
   - 模板专属规则
   - 禁止事项
   - 输出风格与风险提示偏好

### 3.2 不做什么

第一阶段不做：

- 不开放底层 system prompt 编辑。
- 不承诺资料库可让大模型“一模一样”复刻任意输出。
- 不把资料库内容直接作为 authoritative state。
- 不在前端设置页里发明 ready 状态或业务判断。
- 不在 xiaocai 复制 FLARE runtime / session / transport 能力。

## 4. 资料库能力预期

资料库可以稳定增强以下能力：

- 根据公司政策回答采购约束。
- 根据品类知识推荐缺失字段。
- 基于历史案例生成相似结构的初稿。
- 给需求分析、寻源、供应商对比提供上下文。
- 为模板变量提供可引用来源。

资料库不应单独承担以下保证：

- 每次输出格式完全一致。
- 所有字段永远不遗漏。
- 完全复刻历史文档措辞。
- 自动替代字段校验、模板 schema 和人工确认。

强约束输出必须依赖：

```text
字段 schema + 模板结构 + 资料库检索 + AI 规则 + 输出校验 + 渲染器
```

## 6. 数据与执行链路

建议链路：

```text
配置输入
→ xiaocai contract normalize
→ 品类 / 模板 / 资料库 / AI 规则组合
→ FLARE 执行或 LLM 调用
→ provider output normalize
→ schema 校验
→ workbench projection / 文档产物
```

关键原则：

- 配置是产品输入，不是底层 runtime 状态。
- provider 输出必须先 normalize，再进入 xiaocai contract。
- workbench projection 只展示结果，不反向成为真状态。
- 模板结构和字段依赖必须可追溯。

## 7. MVP 范围

### 7.1 P0 MVP

P0 只做最小可用闭环：

1. 品类配置
   - 新增 / 编辑一级与二级品类。
   - 配置必填字段、推荐字段、默认追问。
   - 配置分析维度与寻源偏好。

2. 模板配置
   - 维护模板名称、类型、章节结构。
   - 维护模板变量与字段依赖。
   - 区分草稿态与确认态模板。

3. AI 规则配置
   - 维护全局规则、品类规则、模板规则。
   - 维护禁止事项。
   - 不开放底层 system prompt。

4. 预览测试
   - 输入一段采购需求。
   - 选择品类与模板。
   - 预览追问、分析结构和缺口清单。

### 7.2 P1 增强

- 资料库上传与检索引用。
- 历史案例引用。
- 模板版本管理。
- 配置发布 / 草稿 / 回滚。
- 样例集回归测试。

### 7.3 P2 增强

- 自动从历史采购单沉淀品类规则。
- 自动推荐模板变量。
- 供应商知识图谱。
- 多租户配置继承与覆盖。
- 配置效果 A/B 对比。

## 9. 验收门禁

P0 完成时必须满足：

- 品类字段与追问规则可配置、可读取、可追溯。
- 模板章节与变量依赖可配置，不存在悬空变量。
- AI 规则有作用域和优先级，不直接暴露 system prompt。
- 预览结果能展示缺失字段、下一步动作、分析结构。
- 资料库引用结果必须标识来源，不直接成为领域真状态。
- 失败或信息不足时必须返回缺口，不生成伪完整结果。
