# xiaocai 第一阶段领域范围（Domain Scope）

## 范围原则
- 先做成熟且有业务锚点的场景，不做全覆盖
- 先沉淀可复用资产，再扩场景
- 先确保输出可执行，再考虑体验扩展

## 场景优先级

## P0 场景

### 1) 活动采购
**业务目标**
- 快速完成活动类需求结构化
- 给出可执行的供应商筛选与比选建议
- 输出可落地的活动采购分析与执行建议

**为什么先做**
- 业务材料已明确为当前最成熟场景
- 已有稳定七段式分析结构可直接标准化
- 供应商筛选维度较完整，可直接进入评分资产沉淀

**需要产出的 domain assets**
- 活动场景字段分层（必填/推荐/可选）
- 活动补问策略
- 七段式分析模板
- 活动供应商筛选与评分框架
- 活动场景 RFX 路由建议规则
- 活动输出产物模板

### 2) 礼品定制
**业务目标**
- 区分浅度/深度定制，避免错误采购动作
- 形成礼品采购的约束补齐与风险前置
- 输出可直接执行的供应商筛选与 RFX 建议

**为什么先做**
- 与活动场景并列为当前最可落地方向
- 决策因子清晰（预算、定制深度、数量、交期等）
- 便于形成标准化模板并复用到后续场景

**需要产出的 domain assets**
- 礼品定制深度判定规则
- 礼品字段体系与补问策略
- 礼品分析模板
- 礼品寻源规则与 red flags
- 礼品供应商评分框架
- 礼品输出产物模板

## P1 场景

### 3) 广告 / 拍摄 / 媒体
**业务目标**
- 在 P0 稳定后复用方法论扩展到内容型服务采购
- 引入创意能力与执行能力并重的评估模板

**为什么放 P1**
- 虽属重点品类，但变量多、抽象度高
- 需在 P0 先沉淀“字段-分析-评分-RFX”闭环后再扩展

**需要产出的 domain assets（P1）**
- 内容生产型服务字段包
- 创意/执行双维评分模板
- 合同与版权风险检查清单

### 4) 顾问咨询
**业务目标**
- 支持咨询类项目的范围界定与交付评估
- 输出可比选的方案型采购建议

**为什么放 P1**
- 交付物抽象，质量判断强依赖方法论与案例
- 需要先复用 P0 框架，再补充咨询特有标准

**需要产出的 domain assets（P1）**
- 咨询场景问题流
- 交付里程碑与验收模板
- 咨询供应商评估框架

## 暂缓内容（本阶段不做）
- 全品类采购一次性覆盖
- 自建通用流程引擎
- UI 优先的多场景工作台扩展
- 自动下单/合同履约系统化能力

## 输入 / 输出 / 依赖 / 暂不做

### 输入
- 已确认业务锚点与现有采购材料

### 输出
- 分阶段场景清单（P0/P1）
- 各场景对应 domain assets 交付要求

### 依赖
- P0 文档资产先完整，再进入开发

### 暂不做
- 跨场景统一评分模型“先验一刀切”
