# xiaocai 收敛路线（正式执行版）

## 1. 文档目的

定义后续收敛阶段，保证各仓库执行顺序一致。

---

## 2. 阶段定义

### Phase 1：正式基线对齐（已启动）

目标：

1. `architecture/01-07` 成为唯一正式架构口径。
2. discussions 非基线内容吸收后归档。
3. 所有仓库按统一主链路和模块职责执行。

验收：

1. 正式章节连续叙事完整。
2. 吸收来源可追溯。

### Phase 2：实现面口径统一

目标：

1. 各仓库字段语义、阶段命名、模块边界对齐。
2. 任务中心与交付口径统一。

验收：

1. 关键字段口径在仓库间无冲突。
2. 交付评审可按统一口径执行。

### Phase 3：治理闭环稳定

目标：

1. context、记忆、检索、复盘形成稳定治理闭环。
2. 架构与交付版本保持同步更新。

验收：

1. 复盘与归档可追溯。
2. 架构文档与执行结果持续一致。

---

## 3. 执行约束

1. 先对齐口径，再做实现扩展。
2. 不以临时讨论内容替代正式章节。
3. 每轮收敛都必须可追溯到正式文档更新。
