# Kernel Runtime Engines（正式职责）

## 1. 文档目的

定义 Kernel Runtime 的正式职责边界，并与采购业务闭环逐一对应。

---

## 2. Runtime 总体定位

Kernel Runtime 是平台公共运行层，负责稳定支撑业务链路，不直接承载业务项目装配。

---

## 3. Engine 职责边界

### 3.1 Data Ingestion Engine

负责：

1. 数据源接入与同步任务管理。
2. 文件导入、外部接口接入、重试和状态跟踪。

对应业务环节：

- 需求梳理输入接入
- 外部资料和供应商信息接入

### 3.2 Context / Knowledge Engine

负责：

1. 结构化字段上下文组织。
2. 检索对象组织与按需注入。

对应业务环节：

- 需求分析上下文
- RFX策略上下文
- 寻源检索上下文

### 3.3 Memory Engine

负责：

1. 可复用历史结论和偏好沉淀。
2. 复用对象生命周期管理。

对应业务环节：

- 历史需求复用
- 历史寻源复用

### 3.4 Observation / Review Engine

负责：

1. 关键过程观测、回溯与复盘记录。
2. 对结果可解释性与治理审计提供支撑。

对应业务环节：

- 验收追溯
- 质量复盘

### 3.5 Presentation Engine

负责：

1. 输出结构标准化。
2. 结果视图一致性与交付结构管理。

对应业务环节：

- 报告输出
- 对比清单输出
- 归档输出

---

## 4. 与业务层的约束

1. Runtime 不定义项目业务策略。
2. Runtime 不替代决策路由。
3. Runtime 只提供稳定运行与数据承载能力。
