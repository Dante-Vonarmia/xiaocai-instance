# Decision / Execution / Skills（正式职责）

## 1. 文档目的

定义决策与执行主链在项目闭环中的正式分工。

---

## 2. 核心结论

1. Decision Engine 负责“做什么、按什么顺序做”。
2. Execution Engine 负责“把决策动作稳定执行出来”。
3. Skill 是动作链中的能力单元，不作为架构层替代品。

---

## 3. Decision Engine 职责

1. 任务识别与路由。
2. 阶段判断（需求梳理/分析/RFX/寻源/输出）。
3. 执行动作选择（包括Skill调用选择）。
4. 多动作编排顺序与失败回退策略。

对应业务闭环：

- 决定当前需求进入哪个环节
- 决定何时触发分析、何时触发寻源、何时进入输出

---

## 4. Execution Engine 职责

1. 执行动作调度。
2. 执行状态管理与重试。
3. 输出事件和结果回传。

对应业务闭环：

- 把分析、检索、筛选、输出动作稳定落地

---

## 5. Skills 职责

1. 承载原子业务能力（例如字段提取、规则校验、结果整形等）。
2. 在决策与执行链上被调用，不反向定义架构分层。

---

## 6. 强制约束

1. 不以Skill替代Decision职责。
2. 不以Execution承载业务产品定义。
3. 不在单一Skill中堆叠完整业务闭环。
