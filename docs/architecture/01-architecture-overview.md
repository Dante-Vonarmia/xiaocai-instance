# 小采架构总纲（正式基线）

## 1. 文档目的

本文档是小采项目的正式总纲，用于统一所有仓库与协作角色的执行基线。

本总纲只定义：

1. 本期目标与边界
2. 业务闭环与模块职责
3. 架构主干分层
4. 验收口径与对齐原则

不包含实现细节、伪代码和示例代码。

---

## 2. 本期目标与边界

### 2.1 本期目标

1. 形成采购业务可执行闭环：需求梳理 -> 需求分析 -> RFX策略 -> 智能检索 -> 智能寻源 -> 输出与归档。
2. 形成统一项目容器（Project）与统一任务语义（任务中心字段口径）。
3. 形成可追溯的知识沉淀与复用路径。

### 2.2 本期明确不做

1. 语音输入与语音交互。
2. 法务最终审查替代能力。
3. 未确认的新模块扩展。

---

## 3. 正式业务主链路

统一主链路如下：

`Project立项 -> 需求梳理 -> 需求分析 -> RFX策略 -> 智能检索 -> 智能寻源 -> 结果评估 -> 输出交付 -> 知识沉淀 -> 下次复用`

各仓库与模块必须围绕该链路建设，不再并行维护替代主链路。

---

## 4. 正式模块职责

### 4.1 需求梳理

负责把输入需求转为结构化任务基础信息，作为后续分析和寻源唯一入口。

### 4.2 需求分析

负责形成可评审的分析结论与优先级判断，作为RFX与寻源的上游约束。

### 4.3 RFX策略

负责确定RFI/RFQ/RFP/RFB路径、评估维度与权重、里程碑和发布约束。

### 4.4 智能检索与智能寻源

负责在约束条件下形成候选集、风险提示和对比结果。

### 4.5 输出与归档

负责交付报告、清单、策略结论，并沉淀到可复用知识对象。

### 4.6 知识库与记忆

负责把可复用的案例、规则、来源与结论形成长期资产，不保存未确认中间猜测。

---

## 5. 架构主干

正式架构采用：

1. Project Agent（项目装配层）
2. Kernel Runtime Engines（公共运行层）
3. Decision / Execution Engines（决策执行层）
4. Skills（可插拔能力单元）

详细职责见：

- `02-kernel-runtime-engines.md`
- `03-decision-execution-and-skills.md`
- `04-project-agent-and-subagents.md`

平台边界专项约束见：

- `07-flare-instance-boundary.md`

---

## 6. 验收口径（架构级）

本期按以下口径验收：

1. 闭环完整：从需求梳理到输出归档可串通。
2. 职责清晰：Project、Kernel、Decision/Execution、Skill 不混层。
3. 结果可追溯：关键决策、来源与输出可定位。
4. 术语一致：与 `docs/discussions/requirements-baseline` 不冲突。

---

## 7. 对齐原则

1. 合同基线优先：冲突内容以最终合同口径覆盖。
2. 正式文档优先：`docs/architecture` 为唯一正式架构基线。
3. 讨论文档吸收后归档：已吸收内容不再双轨维护。
