# xiaocai 对齐状态（当前实现 -> 正式基线）

## 1. 文档目的

给出当前实现与正式架构基线的对齐状态，作为各仓库统一执行依据。

---

## 2. 对齐对象

本轮对齐对象来自已吸收讨论内容：

1. 项目交付边界与主流程
2. 产品模块（需求分析/寻源/知识库/会员）
3. 技术治理（context/知识记忆/演进路线）
4. 阶段结论中的有效部分

---

## 3. 当前对齐状态

### 3.1 已统一

1. 正式主链路与模块职责。
2. Project/Kernel/Decision/Execution/Skill 的分层口径。
3. 输出与归档作为闭环必备阶段。
4. 知识沉淀与复用作为正式能力而非附属能力。

### 3.2 仍需持续收敛

1. 任务中心字段口径与各仓库配置一致性。
2. 评估维度与权重口径一致性。
3. 检索/记忆注入约束在各实现面的统一落实。
4. 观测与复盘口径在交付过程中的统一追溯。

---

## 4. 使用规则

1. 各仓库新增能力必须映射到正式模块职责。
2. 与总纲冲突的历史口径不得继续扩散。
3. 讨论内容需先吸收进正式章节再执行。
