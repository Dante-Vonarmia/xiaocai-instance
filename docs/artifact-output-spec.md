# xiaocai P0 产物输出规范（Artifact Output Spec）

## 产物清单（P0）
1. 需求梳理摘要
2. 结构化需求单
3. 需求分析文档
4. 供应商筛选清单
5. 供应商评分表
6. shortlist 推荐
7. RFX 建议
8. 澄清问题单
9. 风险提示单

---

## 1) 需求梳理摘要
- 用途：给需求方快速确认边界
- 输入字段：项目名称、采购目的、场景、预算、交期、地点、品类
- 输出结构：摘要 + 已确认项 + 缺失项 + 下一步
- 消费者：业务方、采购执行
- 是否可直接给业务方：是

## 2) 结构化需求单
- 用途：作为后续分析/寻源/RFX统一输入
- 输入字段：场景必填 + 推荐字段
- 输出结构：字段化清单（必填/推荐/可选）+ 完整度
- 消费者：kernel、operator、sourcing flow
- 是否可直接给业务方：是

## 3) 需求分析文档
- 用途：形成采购分析与执行建议
- 输入字段：结构化需求单 + 约束 + 历史参考
- 输出结构：分场景分析模板（活动七段式/礼品结构）
- 消费者：采购负责人、管理者
- 是否可直接给业务方：是（需采购确认）

## 4) 供应商筛选清单
- 用途：展示 longlist 初筛结果
- 输入字段：寻源约束、供应商基础资料
- 输出结构：供应商列表 + 准入/淘汰原因
- 消费者：sourcing flow、operator
- 是否可直接给业务方：可

## 5) 供应商评分表
- 用途：形成可解释比选依据
- 输入字段：评估维度、供应商证据、评分规则
- 输出结构：维度分、总分、风险标签
- 消费者：采购评审、管理者
- 是否可直接给业务方：是

## 6) shortlist 推荐
- 用途：给出建议入围名单
- 输入字段：评分表 + 风险复核
- 输出结构：推荐名单 + 推荐理由 + 风险提示 + 备选
- 消费者：采购决策方
- 是否可直接给业务方：是

## 7) RFX 建议
- 用途：建议下一步采购动作
- 输入字段：项目类型、目标、预算、约束、完整度
- 输出结构：推荐动作 + 理由 + 缺口 + 下一步
- 消费者：采购执行、决策方
- 是否可直接给业务方：是

## 8) 澄清问题单
- 用途：补齐关键缺失字段
- 输入字段：当前字段状态、readiness规则
- 输出结构：问题列表（优先级、原因、影响）
- 消费者：需求方、采购执行
- 是否可直接给业务方：是

## 9) 风险提示单
- 用途：提前暴露关键风险
- 输入字段：分析结果、供应商风险、约束冲突
- 输出结构：风险项 + 等级 + 建议动作
- 消费者：采购负责人、管理者
- 是否可直接给业务方：是（建议附解释）

## 输入 / 输出 / 依赖 / 暂不做

### 输入
- domain pack 输出的结构化结果

### 输出
- 可交付、可复核、可追溯的标准产物

### 依赖
- activity/gift pack
- supplier evaluation framework
- rfx routing spec

### 暂不做
- 复杂排版/品牌化视觉模板
