# 05 Analysis And Document Templates

## 边界

- 本文定义 analysis 与 RFX 文档输出规则。
- 输出必须绑定字段模型与阶段状态。

## 标准

### 1. Requirement Analysis

- analysis 依赖字段集合必须明确列出。
- 缺关键字段时禁止生成完整分析，必须返回缺口清单。
- analysis 输出采用结构化章节，不允许完全 freestyle。

### 2. RFX Templates

- 模板范围: RFI / RFQ / RFP / RFB。
- 每个模板必须定义:
- 必要字段依赖
- 可选字段依赖
- 变量绑定规则
- 草稿态与确认态区别

### 3. 文档归档

- 输出文档必须可归档到 Project。
- 输出记录要关联字段快照与生成时间。

## 验收

- 缺关键字段时系统明确说明缺口，不瞎编。
- 模板变量均有来源字段，不存在悬空变量。
- 文档可区分草稿版与确认版。

## 典型反例

- RFX 直接由 prompt 一次生成，无字段依赖校验。
- 关键字段缺失仍输出“完整策略建议”。
- 生成内容不可追溯到 requirement 字段。

## 不做什么

- 不用 prompt 文本代替模板依赖管理。
- 不输出不可归档的临时结论。
