# 08 Domain ADR Index

## 边界

- 本文记录 `xiaocai` 领域侧关键决策。
- 仅记录 procurement domain 决策，不记录平台通用实现细节。

## ADR 模板

- 背景
- 决策
- 备选方案
- 影响
- 后续注意事项

## ADR 列表

### ADR-001 Taxonomy 分层策略

- 背景: 需兼顾可执行与可扩展。
- 决策: 使用 采购负责类 -> 一级品类 -> 二级品类。
- 备选方案: 扁平标签体系。
- 影响: 分类路径清晰，可驱动字段加载。
- 后续注意: 新品类先入配置再入流程。

### ADR-002 Common vs Category-specific 字段边界

- 背景: 避免字段泛化过度或重复。
- 决策: common 保障主链路，specific 由品类占位增补。
- 备选方案: 全部字段统一大表不分层。
- 影响: 降低跨品类耦合与实现复杂度。
- 后续注意: 不允许把 specific 退化成代码分支。

### ADR-003 Search Mapping 结构化优先

- 背景: 检索结果易失控。
- 决策: 强制 target_field + evidence 结构。
- 备选方案: 纯文本搜索摘要。
- 影响: 可回填、可验收、可追溯。
- 后续注意: 无 target_field 的请求默认不执行广域检索。

### ADR-004 Replace Rule 分级策略

- 背景: 替换安全与效率冲突。
- 决策: auto/recommend/confirm 三层替换策略。
- 备选方案: 全部自动替换或全部人工确认。
- 影响: 关键字段安全性提升。
- 后续注意: replace history 必须落盘可追踪。

### ADR-005 Workflow 顺序约束

- 背景: 先检索后梳理导致结果噪声高。
- 决策: 先梳理再检索，再回填再分析再文档输出。
- 备选方案: 并行或自由跳步。
- 影响: 降低错误推荐与无关信息。
- 后续注意: blocker 时允许回退，不允许跳过关键阶段。

### ADR-006 平台复用优先

- 背景: 避免在 instance 重造平台能力。
- 决策: workspace/contracts/storage/orchestration 优先复用 FLARE。
- 备选方案: xiaocai 内自建运行时层。
- 影响: 降低维护成本，减少架构漂移。
- 后续注意: 仅在平台能力无法承接采购语义时提增量需求。

## 验收

- 每个新增领域关键决策都能落一条 ADR。
- ADR 变更必须同步影响到相关标准文档。

## 不做什么

- 不把 ADR 写成会议纪要。
- 不接受“凭感觉”的无背景决策。
