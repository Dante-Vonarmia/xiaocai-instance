# 10 xiaocai Business Capability Acceptance Baseline

最后更新: 2026-04-13

## 边界

- 本文用于定义 `xiaocai` 业务能力验收基准，不替代部署连通性验收。
- 本文聚焦“采购业务闭环能力”，不评价底层模型优劣。
- 本文中的“现状判定”是当前基线，用于后续迭代对比。

## 能力地图（当前基线）

| 业务能力 | 当前判定 | 差距等级 |
| --- | --- | --- |
| 需求入口（chat run/stream） | 已实现 | 小 |
| 身份与会话归属 | 已实现 | 小 |
| 需求结构化（可执行采购单） | 半实现 | 中 |
| 智能询源（供应商/渠道发现） | 半实现 | 中-大 |
| 比价评估（价格/交期/质量/风险） | 半实现 | 大 |
| 推荐决策（推荐+理由） | 半实现 | 大 |
| 采购执行（询价/下单/合同） | 缺失或弱实现 | 大 |
| 履约跟踪（发货/到货/异常） | 缺失或弱实现 | 大 |
| 对账复盘（成本/时效/节省） | 缺失或弱实现 | 大 |
| 知识沉淀复用 | 半实现 | 中 |
| 权限审计与审批轨迹 | 半实现 | 中-大 |
| 业务 KPI 自动采集 | 缺失或弱实现 | 大 |

## 验收等级（业务导向）

### L1（可用对话）

1. 能完成身份交换并建立会话。
2. `chat/run` 与 `chat/stream` 可稳定返回结果。

### L2（可执行需求）

1. 聊天输入可落为结构化需求对象（字段完整、可追溯）。
2. 结构化结果可被后续流程直接消费，不依赖人工二次整理。

### L3（可用采购建议）

1. 能给出候选供应来源、评估维度和可解释推荐理由。
2. 比价与评分规则可复现，同一输入在同版本下结果可解释。

### L4（业务闭环）

1. 从需求到执行、履约、复盘全链路可追踪。
2. 关键业务 KPI 可自动产出并复核。

## 验收条目（达标线）

| 能力项 | 达标线 | 证据要求 |
| --- | --- | --- |
| 需求结构化 | 至少 5 个核心采购场景可稳定结构化 | 输入样例 + 结构化输出 + 字段断言 |
| 询源与评估 | 每个场景至少 3 个候选来源并附评估维度 | 来源列表 + 评分明细 + 推荐理由 |
| 推荐决策 | 推荐结果可追溯到规则与证据 | 规则版本 + 证据引用 + 审核记录 |
| 执行履约 | 至少 1 条主链路可贯通到状态完结 | 状态流转记录 + 异常处理记录 |
| 经营复盘 | 输出周期、成本、节省、命中率指标 | 指标报表 + 采集口径说明 |

## 当前验收结论（基线）

1. 当前可认定达到 `L1`。
2. `L2/L3` 有基础能力但未达到稳定验收线。
3. `L4` 尚未形成业务闭环验收证据。

## 不做什么

- 不把“回答看起来不错”作为通过标准。
- 不把单次 demo 成功视为业务能力达标。
- 不以部署成功替代业务闭环验收。
