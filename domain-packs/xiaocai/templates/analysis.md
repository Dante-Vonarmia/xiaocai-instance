# 需求分析动态章节规则

本文件只定义分析输出的展示原则，不固定报告章节模板。

- 根据用户请求、已确认需求、候选字段、资料证据和当前采购阶段自然组织章节。
- 使用清晰的 Markdown 标题层级；章节应能被目录定位、隐藏和后续编辑。
- 可按需覆盖摘要、背景、需求边界、规格与假设、市场与供应、成本、风险、采购策略、供应商建议、RFX 路径和执行计划等内容，但不要机械输出空章节。
- 已确认事实、当前假设、候选信息、待验证信息必须区分表达。
- 缺失信息后置到必要的待补充/待验证章节，不要用大量占位符填满正文。
- 表格、列表和编号可按内容需要使用；优先保证 Markdown 清晰可读。
- 不输出内部 workflow、action key、debug 状态或实现细节。
